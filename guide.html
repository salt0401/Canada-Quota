1. What Does This Tool Do?

This tool automatically monitors Canadian steel import quota (TRQ) utilization every Monday and produces an Excel spreadsheet with:

  • TRQ utilization data — how much of each product’s quota has been used, broken down by country
  • Weekly time series — each Monday adds a new column, so you can see how utilization progresses over the quarter
  • B1 import data — actual customs import volumes by product and country (when available)
  • OVER flag — highlights when a country hits its maximum share cap

  • Smart Automatically detects quarterly transitions and creates new sheets when a new quarter begins

Important: TRQ vs. B1 Data

The spreadsheet contains two types of data that measure different things:

TRQ SheetsB1 Sheet
MeasuresImport permits issuedActual customs entries
QuarterOffset (Mar 26 – Jun 26, etc.)Calendar (Apr 1 – Jun 30, etc.)
Update speedNear real-timeSeveral days lag

There is a ~5-6 day boundary mismatch between the two quarter systems. This is noted in the B1 sheet.

5. Setup Instructions (One-Time)

Lin will provide you with a .zip file containing all project files. Follow these steps:

1
Unzip the Project

Extract the .zip file to any location (e.g., your Desktop). You should see a folder called Canada-Quota with files inside.

2
Create a New GitHub Repository

Go to github.com/new:

  • Repository name: Canada-Quota
  • Set to Public
  • Do NOT check “Add a README file”
  • Click Create repository

After creating, GitHub will show a page with setup commands. Keep this page open — you’ll need the URL (it looks like https://github.com/YOUR-USERNAME/Canada-Quota.git).

3
Edit setup.bat

Open setup.bat in the unzipped folder with a text editor (right-click → Edit or Open with Notepad). Find this line:

set REPO_URL=https://github.com/YOUR-USERNAME/Canada-Quota.git

Change YOUR-USERNAME to your actual GitHub username. Save the file.

4
Upload Files to GitHub

Go to your new repo page on GitHub. Click “uploading an existing file” link. Drag and drop all files and folders from the unzipped folder into the upload area. Click Commit changes.

Make sure to include the hidden .github folder and .gitignore file. On Windows, you may need to enable “Show hidden files” in File Explorer (View → Show → Hidden items).
5
Enable GitHub Actions

In your repo, click the Actions tab. GitHub may ask you to enable workflows — click “I understand my workflows, go ahead and enable them”.

6
Run It Manually (First Time)

In the Actions tab, click “Weekly Canada TRQ Update” on the left, then click “Run workflow” → “Run workflow” (green button). Wait 1-2 minutes for it to complete (green check mark).

7
Clone the Repo to Your Computer

Double-click setup.bat (the one you edited in step 3). This will download the repo to your Desktop as a folder called Canada-Quota.

If prompted to log in, enter your GitHub username and a Personal Access Token (not your password). To create a token: github.com/settings/tokens/new → check “repo” → Generate token → copy and paste it as the password.

8
Done!

From now on, the script runs automatically every Monday. To get the latest Excel file, just double-click update.bat inside the Canada-Quota folder on your Desktop.

7. FAQ

What if the script stops working?

GitHub sends you an email when a workflow fails. Common causes:

  • The Canadian government changed their website — the script may need updating
  • The CSV format changed — contact Lin for an update

What happens when a new quarter starts?

The script automatically detects the new quarter and creates new sheets (e.g., non-FTA Q1, FTA Q1). The previous quarter’s sheets remain in the workbook as historical records.

Can I run it more than once a week?

Yes! Go to Actions → Weekly Canada TRQ Update → Run workflow at any time. Then double-click update.bat to get the new data.

Why are some B1 products showing no data?

B1 data uses calendar quarters (Jan–Mar, Apr–Jun, etc.) while TRQ uses offset quarters (Mar 26–Jun 26, etc.). At the start of a new TRQ quarter, B1 data for the matching calendar quarter may not be available yet.

Is this free?

Yes. GitHub Free accounts include 2,000 minutes/month of Actions time. This script uses ~8 minutes/month. That’s less than 1% of the free quota.

What does update.bat actually do?

It runs a single command: git pull, which downloads any new files from GitHub to your local folder. Then it opens the Excel file. It does not upload or change anything.
