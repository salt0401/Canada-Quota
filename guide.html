1. What Does This Tool Do?

This tool automatically monitors Canadian steel import quota (TRQ) utilization every Monday and produces an Excel spreadsheet with:

  • TRQ utilization data — how much of each product’s quota has been used, broken down by country
  • Weekly time series — each Monday adds a new column, so you can see how utilization progresses over the quarter
  • B1 import data — actual customs import volumes by product and country (when available)
  • OVER flag — highlights when a country hits its maximum share cap

Important: TRQ vs. B1 Data

The spreadsheet contains two types of data that measure different things:

TRQ SheetsB1 Sheet
MeasuresImport permits issuedActual customs entries
QuarterOffset (Mar 26 – Jun 26, etc.)Calendar (Apr 1 – Jun 30, etc.)
Update speedNear real-timeSeveral days lag

There is a ~5-6 day boundary mismatch between the two quarter systems. This is noted in the B1 sheet.

4. Setup Instructions (One-Time)

Follow these steps to get the automation running on your own GitHub account:

1
Create a GitHub Account

Go to github.com/join and sign up for a free account. Use your work email if you prefer.

2
Import the Repository

Go to github.com/new/import and paste this URL:

https://github.com/salt0401/Canada-Quota.git

Name your repo (e.g., Canada-Quota), set it to Private (recommended), and click Begin import.

3
Enable GitHub Actions

In your new repo, click the Actions tab. GitHub may ask you to enable workflows — click “I understand my workflows, go ahead and enable them”.

4
Run It Manually (First Time)

In the Actions tab, click “Weekly Canada TRQ Update” on the left, then click “Run workflow” → “Run workflow” (green button). Wait about 1-2 minutes for it to complete.

5
Update download_latest.bat

Open the file download_latest.bat in a text editor (right-click → Edit). Find the line with salt0401/Canada-Quota and change it to your GitHub username and repo name. For example:

https://github.com/YOUR-USERNAME/Canada-Quota/releases/latest/download/canada_trq_tracker.xlsx

Save the file and place it somewhere convenient (e.g., your Desktop).

6
Done!

From now on, the script runs automatically every Monday. To get the latest Excel file, just double-click download_latest.bat.

If you set the repo to Private, the download .bat file won’t work without authentication. In that case, set the repo to Public, or download the file manually from the GitHub Releases page in your browser.

5. How to Get the Latest Data

Option A: Double-click the .bat file Easiest

  1. Double-click download_latest.bat
  2. The latest Excel file appears on your Desktop

6. FAQ

What if the script stops working?

GitHub sends you an email when a workflow fails. Common causes:

  • The Canadian government changed their website — the script may need updating
  • The CSV format changed — contact Lin for an update

What happens when a new quarter starts?

The script automatically detects the new quarter and creates new sheets (e.g., non-FTA Q1, FTA Q1). The previous quarter’s sheets remain in the workbook as historical records.

Can I run it more than once a week?

Yes! Go to Actions → Weekly Canada TRQ Update → Run workflow at any time. If the data hasn’t changed since the last run, it won’t add a duplicate column.

Why are some B1 products showing no data?

B1 data uses calendar quarters (Jan–Mar, Apr–Jun, etc.) while TRQ uses offset quarters (Mar 26–Jun 26, etc.). At the start of a new TRQ quarter, B1 data for the matching calendar quarter may not be available yet. It will appear once the calendar quarter begins.

Is this free?

Yes. GitHub Free accounts include 2,000 minutes/month of Actions time. This script uses ~8 minutes/month (2 min × 4 Mondays). That’s less than 1% of the free quota.
